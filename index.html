HTML

  • The head element gives information about the webpage that the user cannot see.
  • The body element contains the items the user can see.
  • The title element defines the title for the webpage.
  • The style element defines CSS styles.
  • The base element defines the base URL for a page.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadeta. (e.g. charset, http-equiv, viewport)
  • The script element embeds client-side scripts in the HTML document.
  • The div element creates a division in the page.
  • HTML Elements Reference Sheet

CSS

  • Margin adds space around the outside of an element.
  • Padding adds space around content inside of an element.
  • Selector defines element/attributes which the rules/declarations will apply.
  • CSS Properties/Class/Etc. Reference Sheet

Git

  • A Repository (repo) is a folder in which all files and their version histories are stored.
  • A Branch is a workspace in which you can make changes that won't affect the live site.
  • Markdown (.md) is a way to write in Github that converts plain text to Github code.
  • Commit changes refers to saving a record of a change made to a file in a repo.
  • A Pull Request is the way to ask for changes made to a branch to be merged into another branch that also allows for multiple users to see, discuss, and review work being done.
  • Merge refers to what happens after a pull request is approved, where the commit will be pulled in (merged) from one branch to another and then deployed on the live site.
  • Issues are how work is tracked when using Git. Allow users to report new tasks and content fixes, as well as allows users to track progress on a project board from beginning to end.
  • Federalist is a platform that securely deploys a website from a Github repo in minutes and lets users preview proposed and published changes.
  • Clone refers to a copy of a repo hosted locally on the user's device.
  • Fork refers to when you copy another user's repo to your own account and make changes that leave the original unaffected (but still allows you to update).
  • Push/Pull merging changes to and from local, respectively.
  • Git Cheat Sheet

JavaScript

  • Variables are containers that store values.
  • Comments are snippets of non-visible text that are surrounded by "/* */".
  • Conditionals are code structures used to test if an expression returns true or not.
  • Functions are a set of instructions that tells the computer how to perform a task.
  • Events are code structures that listen for activity in the browser and run code in reponse.
  • Javascript Reference Sheet
